# Physical Model Spec

## 组件层级

当前物理模型按以下层级组织：

1. 桌面。
2. 卡片盒。
3. 抽屉。
4. 抽屉内容根节点。
5. 卡片和分隔片。
6. 可视几何体与碰撞体。

系统物理状态分析属于本文件，不属于主控层。后续视频分析会不断补充和修正这里。

## 桌面规格

- 桌面是临时摊开和整理卡片的工作平面。
- 桌面包含卡片盒区域、桌面摊开区域和导入落点区域。
- 桌面不理解卡片内容，只承载位置和摆放关系。

## 卡片盒规格

- 当前 Demo 使用几何体表示卡片盒外壳。
- 卡片盒包含顶部、底部、侧壁和背板等可见结构。
- 卡片盒用于承载多个抽屉。
- 后续可替换更精细模型，但不应改变保存边界。

## 抽屉规格

- 当前 Demo 至少包含 3 个抽屉。
- 抽屉包含地板、侧壁、前挡边和把手。
- 抽屉沿固定轴开合，并具有最小和最大开合范围。
- 抽屉内有内容根节点，用于承载卡片和分隔片。

## 卡片规格

- 卡片是有厚度的 3D 薄长方体。
- 卡片包含正面、背面、边缘和碰撞体。
- 正面可显示占位贴图或导入图片。
- 卡片拥有位置、旋转、所在区域、图片资源路径和简单物理状态。

## 分隔片规格

- 分隔片是实体薄片的虚拟对应物。
- 分隔片可以比普通卡片略高或使用不同颜色。
- 分隔片可被拿起、移动、放下和放入抽屉。
- 分隔片不是软件标签，不应出现标签编辑、搜索或分类入口。

## 碰撞与选中规则

- 可交互对象应具有碰撞体。
- 选中对象应有可观察的视觉提示。
- 当前操作对象可标记为 active，其余对象可保持 sleeping。
- 抽屉使用受限运动，不使用完全自由刚体模拟。

## 抽屉内排列关系

- 当前 Demo 使用简化吸附排列。
- 放入抽屉的卡片应保持在抽屉内部并随抽屉移动。
- 后续大量卡片场景应优先使用对象池、窗口化可见范围和静态堆叠表示。

## 待视频观察确认的问题

- 真实卡片在抽屉中的倾斜、密度和可见边缘。
- 用户浏览一组卡片时的手部动作顺序。
- 分隔片的真实尺寸、摆放方式和使用频率。
- 桌面摊开区域和抽屉之间的常见距离关系。

当前没有假装已经完成 Scheper 视频分析。

## 实体物理组件复原（CMD-2026-06-20-001）

时间：2026-06-21 10:54 +08:00

### 桌面

- 桌面已改为连续木质工作面。
- 移除了原先 `CardBoxZone`、`WorkSpreadZone`、`ImportLandingZone` 等固定功能区视觉。
- 桌面保留自然木纹、前后边缘和可自由摆放的平面关系。
- 桌面不承载阅读区、写作区、待归档区、索引区等软件式固定分区。

### 纸质卡片

- 通用卡片脚本仍为 `Card.gd`，不新增 Main Card、Bibcard、Index Card 等功能性语义类型。
- 卡片差异通过物理 profile 表达：`plain_paper`、`ruled_paper`、`thin_plain_paper`、`colored_paper`、`divider_tab`。
- 每张卡片包含可见厚度、正面、背面、边缘、纸张颜色、地址/编号占位和纸面墨迹占位。
- 横线纸和有色纸通过材质与表面线条表达，不作为知识分类。
- 绿色链接标记和红色外部引用标记只是纸面视觉占位，不进入搜索、标签或图谱系统。

### 卡片盒 / notebox

- 卡片盒脚本 `CardBox.gd` 已改为木质 notebox 表现。
- 盒体包含顶部、底部、左右侧板、背板、前框、内部滑轨和木纹占位。
- 抽屉前方包含把手和铭牌槽；铭牌槽只是实体部件，不是软件目录。

### 抽屉与滑轨

- 抽屉脚本 `Drawer.gd` 使用约束式单轴滑动。
- 抽屉有关闭、半开、全开状态，并在最大拉出距离处停止。
- 抽屉包含底板、侧壁、后挡板、前面板、把手和左右滑轨。
- 抽屉打开时露出内部纸卡队列；关闭时盒体遮蔽队列。

### 抽屉内纸卡队列

- 抽屉内卡片改为站立或轻微倾斜的纸卡队列。
- 队列按照 `drawer_slot_index` 保存顺序。
- 卡片之间保留小间距，形成密度和缝隙。
- 拖动卡片靠近打开抽屉时显示插入缝隙高亮。
- 松开后卡片插入对应 slot，队列重新排布。

### 分隔片

- 分隔片仍继承通用纸卡行为，但使用 `divider_tab` 物理 profile。
- 分隔片比普通卡更高，并带顶部 tab 和短标签占位。
- 分隔片与普通卡处于同一物理队列。
- 分隔片不是软件标签系统，不提供搜索、分类或树状目录能力。

### 保存状态

保存状态扩展为记录以下物理属性：

- `physical_profile`
- `has_ruled_lines`
- `has_tab`
- `tab_label`
- `has_ink_marks`
- `front_color`
- `back_color`
- `edge_color`
- `drawer_slot_index`
- 卡片所在区域、桌面位置、旋转、翻面状态和抽屉打开程度

保存状态仍只保存物理状态和资源路径，不保存 OCR、标签、自动分类结果、AI 摘要或知识图谱数据。

### 当前限制

- 纸张仍使用硬质薄盒体近似，不做软体纸张仿真。
- 抽屉滑轨使用脚本约束和可控移动，不做真实摩擦仿真。
- 插入缝隙是视觉高亮与吸附点模拟，不是复杂纸堆碰撞。
- 大规模卡片性能优化仍待后续专门任务。

### 人工验收清单

用户通过 UU 远程运行主场景后，应逐步确认：

1. 进入主场景后能看到连续木质桌面，而不是固定功能分区 dashboard。
2. 能看到木质 notebox，包含盒体、抽屉、把手、铭牌槽和滑轨感。
3. 至少一个抽屉处于打开状态，并能看到内部站立纸卡队列。
4. 队列中能看到普通纸卡、横线或彩色纸卡，以及更高的分隔片。
5. 点击并拖动抽屉把手时，抽屉只沿一个方向滑动，并在最大开合处停止。
6. 从打开抽屉中点击纸卡并拖到桌面，纸卡能从队列取出并平放到桌面。
7. 桌面上的纸卡可移动、旋转和翻面。
8. 拖动纸卡回到打开抽屉时，能看到插入缝隙高亮。
9. 松开后纸卡进入队列中的物理位置，队列重新形成有密度的排列。
10. 分隔片可像更高纸片一样被拿出、移动并插回队列。
11. UI 中没有 OCR、搜索、标签、自动分类、AI 摘要或知识图谱入口。
12. 保存/加载后，抽屉开合、桌面卡片位置、卡片翻面状态和抽屉队列顺序能够恢复。
