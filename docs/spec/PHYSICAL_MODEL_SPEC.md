# Physical Model Spec

## 组件层级

当前物理模型按以下层级组织：

1. 桌面。
2. 卡片盒。
3. 抽屉。
4. 抽屉内容根节点。
5. 卡片和分隔片。
6. 可视几何体与碰撞体。

系统物理状态分析属于本文件，不属于主控层。后续视频分析会不断补充和修正这里。

## 桌面规格

- 桌面是临时摊开和整理卡片的工作平面。
- 桌面包含卡片盒区域、桌面摊开区域和导入落点区域。
- 桌面不理解卡片内容，只承载位置和摆放关系。

## 卡片盒规格

- 当前 Demo 使用几何体表示卡片盒外壳。
- 卡片盒包含顶部、底部、侧壁和背板等可见结构。
- 卡片盒用于承载多个抽屉。
- 后续可替换更精细模型，但不应改变保存边界。

## 抽屉规格

- 当前 Demo 至少包含 3 个抽屉。
- 抽屉包含地板、侧壁、前挡边和把手。
- 抽屉沿固定轴开合，并具有最小和最大开合范围。
- 抽屉内有内容根节点，用于承载卡片和分隔片。

## 卡片规格

- 卡片是有厚度的 3D 薄长方体。
- 卡片包含正面、背面、边缘和碰撞体。
- 正面可显示占位贴图或导入图片。
- 卡片拥有位置、旋转、所在区域、图片资源路径和简单物理状态。

## 分隔片规格

- 分隔片是实体薄片的虚拟对应物。
- 分隔片可以比普通卡片略高或使用不同颜色。
- 分隔片可被拿起、移动、放下和放入抽屉。
- 分隔片不是软件标签，不应出现标签编辑、搜索或分类入口。

## 碰撞与选中规则

- 可交互对象应具有碰撞体。
- 选中对象应有可观察的视觉提示。
- 当前操作对象可标记为 active，其余对象可保持 sleeping。
- 抽屉使用受限运动，不使用完全自由刚体模拟。

## 抽屉内排列关系

- 当前 Demo 使用简化吸附排列。
- 放入抽屉的卡片应保持在抽屉内部并随抽屉移动。
- 后续大量卡片场景应优先使用对象池、窗口化可见范围和静态堆叠表示。

## 待视频观察确认的问题

- 真实卡片在抽屉中的倾斜、密度和可见边缘。
- 用户浏览一组卡片时的手部动作顺序。
- 分隔片的真实尺寸、摆放方式和使用频率。
- 桌面摊开区域和抽屉之间的常见距离关系。

当前没有假装已经完成 Scheper 视频分析。
