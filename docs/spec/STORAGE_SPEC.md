# Storage Spec

## 存储原则

项目的大量本地数据应默认放在 D 盘项目路径下，不应把导入图片、缩略图、缓存、备份和世界状态等大量数据放到 C 盘。

当前项目路径：

```text
D:\codex project\VirtualAntinetBox\project\virtual-antinet-box
```

## 建议本地数据目录

后续实现任务应优先使用项目目录下的本地数据结构：

```text
local_data/
├─ world_state/
├─ imported_cards/
├─ thumbnails/
├─ cache/
└─ backups/
```

## 数据类型边界

允许保存：

- 卡片或分隔片 ID。
- 图片资源路径。
- 位置。
- 旋转。
- 所在区域。
- 抽屉开合程度。
- 尺寸和简单物理状态。
- 缩略图和缓存路径。

禁止保存为项目功能的数据：

- OCR 文本。
- 标签。
- 主题。
- 自动链接。
- AI 摘要。
- 知识图谱。
- 根据内容生成的编号或推荐位置。

## 当前实现状态

当前 Demo 的保存路径仍使用 Godot `user://world_state.json`，导入图片也可能进入 Godot 用户目录。后续需要通过独立实现任务改造为 D 盘项目路径下的 `local_data/` 策略。

本轮 `CMD-2026-06-19-002` 只做文档结构重构，不修改保存/加载代码，也不修改导入图片实现。

## 后续改造要求

后续如果创建存储改造 CMD，应明确：

1. 是否迁移现有 `user://` 数据。
2. `local_data/` 是否纳入 Git 忽略。
3. 导入图片复制策略。
4. 缩略图和高清图分层策略。
5. 备份和恢复策略。
