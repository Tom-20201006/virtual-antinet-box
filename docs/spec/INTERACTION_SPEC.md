# Interaction Spec

## 交互设计目标

交互设计应尽量模拟实体卡片盒的基本动作：打开抽屉、浏览卡片、取出卡片、放到桌面、翻看、旋转、插回抽屉和使用分隔片。系统应强调物理动作，而不是内容理解。

## 核心工作流

当前工作流框架：

1. 用户在桌面总览中定位卡片盒。
2. 用户打开抽屉。
3. 用户浏览可见卡片或分隔片。
4. 用户取出卡片或分隔片。
5. 用户在桌面摊开区域进行临时摆放。
6. 用户旋转、翻面或重新摆放卡片。
7. 用户把卡片或分隔片插回抽屉。
8. 用户保存世界状态。

## 实体动作到虚拟动作的映射

- 打开抽屉：鼠标拖动抽屉或按键切换开合。
- 浏览卡片：摄像机移动、缩放和抽屉视角切换。
- 取出卡片：鼠标选择并拖离抽屉。
- 桌面摊开：拖放到桌面工作区域。
- 翻看卡片：按键翻面。
- 调整朝向：按键旋转。
- 插回卡片：拖放到打开抽屉内部并吸附排列。
- 使用分隔片：以普通实体薄片处理，不提供标签编辑入口。

## 必须模拟的动作

- 抽屉沿固定方向开合。
- 卡片或分隔片可选中、拿起、移动和放下。
- 卡片可旋转和翻面。
- 卡片可放到桌面，也可放入打开的抽屉。
- 卡片可从抽屉中取出。
- 摄像机支持总览、抽屉视角和聚焦对象。

## 可以简化的动作

- 抽屉滑轨可用固定轴受限运动表示。
- 抽屉内卡片排列可使用吸附位置，不要求真实纸堆细节。
- 卡片材质可先使用几何体和贴图，不要求真实纸张弯曲。
- 分隔片可复用卡片交互逻辑。

## 暂不实现的动作

- 手部模型。
- VR 操作。
- 真实纸张弯曲。
- 精细木质滑轨摩擦。
- 基于卡片内容的搜索、标签、自动分类或推荐。

## 待视频分析补充的问题

- 浏览卡片时的手部和卡片相对关系。
- 取出卡片后的桌面摊开方式。
- 插回卡片时的位置判断。
- 分隔片在实体流程中的真实用途。
