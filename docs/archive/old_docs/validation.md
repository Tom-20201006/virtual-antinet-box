# Validation

本文档是 VirtualAntinetBox 的验收标准文档。以后新增或修改任何验收步骤，都必须按本文档中的格式编写：先写清前置条件，再写逐步操作，每一步都必须包含可观察的通过标准，不能只写“确认正常”。

## 验收步骤编写标准

后续所有验收步骤均按以下标准编写：

1. 每个验收项必须有明确编号，例如 `VAB-VAL-001`。
2. 每个验收项必须写明前置条件。
3. 每个验收项必须拆成单步操作，避免把多个动作合并成一句话。
4. 每个操作后必须写明可观察的通过标准。
5. 如果涉及保存、加载、导入、拖拽等状态变化，必须说明变化前和变化后的期望状态。
6. 如果涉及禁止功能，必须明确检查没有 OCR、搜索、标签、自动分类、自动链接、AI 摘要、知识图谱等语义功能入口。
7. 验收步骤只验证物理空间、物体、资源路径和交互状态，不验证或推断卡片文字内容。
8. 常见失败情况必须给出排查方向。

## 运行前准备

前置条件：

1. 已安装 Godot 4.6 标准版。
2. 项目目录为 `D:\codex project\VirtualAntinetBox\project\virtual-antinet-box`。
3. 主场景为 `res://scenes/Main.tscn`。
4. 准备一张本地图片用于导入测试，格式为 `png`、`jpg`、`jpeg` 或 `webp`。
5. 测试导入图片只作为卡片贴图使用，不需要检查图片文字内容。

通过标准：

1. Godot 能打开项目目录。
2. 项目配置中主场景指向 `res://scenes/Main.tscn`。
3. 测试人员知道导入图片不会被 OCR、搜索或内容识别。

## VAB-VAL-001 打开项目

前置条件：

1. Godot 4.6 已安装。
2. 项目文件夹存在。

操作步骤：

1. 启动 Godot。
   - 通过标准：Godot 项目管理器或编辑器窗口正常打开。
2. 选择打开项目目录 `D:\codex project\VirtualAntinetBox\project\virtual-antinet-box`。
   - 通过标准：Godot 识别该目录为 Godot 项目。
3. 打开项目后查看文件系统面板。
   - 通过标准：能看到 `scenes/`、`scripts/`、`docs/`、`assets/` 等目录。
4. 查看主场景设置。
   - 通过标准：主场景为 `res://scenes/Main.tscn`。

失败排查：

- 如果 Godot 不识别项目，检查 `project.godot` 是否存在。
- 如果主场景未设置，打开 `project.godot`，确认 `run/main_scene` 指向 `res://scenes/Main.tscn`。

## VAB-VAL-002 运行主场景

前置条件：

1. 项目已在 Godot 中打开。
2. 主场景已设置。

操作步骤：

1. 点击 Godot 的运行项目按钮。
   - 通过标准：游戏窗口打开，没有脚本解析错误。
2. 等待 2 秒。
   - 通过标准：窗口中显示 3D 桌面，而不是空白画面。
3. 观察左上角 UI。
   - 通过标准：能看到“导入”“保存”“载入”“总览”“抽屉”“聚焦”按钮和状态文字。

失败排查：

- 如果窗口空白，按 `1` 或 `Home` 回到总览视角。
- 如果控制台有脚本错误，优先检查 `scripts/Main.gd` 和场景路径。

## VAB-VAL-005 验证抽屉开合

前置条件：

1. 主场景正在运行。
2. 至少一个抽屉可见。

操作步骤：

1. 左键按住抽屉把手。

2. 向下拖动鼠标。
   - 通过标准：抽屉沿固定方向拉开。
3. 继续拖动到最大开合附近。
   - 通过标准：抽屉停止在最大拉出距离内，不会脱离卡片盒。
4. 向上拖动鼠标。
   - 通过标准：抽屉沿同一轴线推回。
5. 选中抽屉后按 `O`。
   - 通过标准：抽屉在打开和关闭状态之间切换。

失败排查：

- 如果抽屉方向不符合预期，确认拖动的是鼠标上下方向。
- 如果抽屉不可见，按 `2` 切换到抽屉视角。

## VAB-VAL-007 验证卡片拿起和移动

前置条件：

1. 主场景正在运行。
2. 至少一张卡片可见。

操作步骤：

1. 左键按住一张卡片。
   - 通过标准：卡片被选中并进入拖拽状态。
2. 移动鼠标到桌面摊开区域。
   - 通过标准：卡片跟随鼠标在桌面上方移动。

   - 通过标准：卡片落在桌面摊开区域。

## 最终验收清单

- 项目能打开。
- 主场景能运行。
- 有完整桌面。
- 有卡片盒。
- 有至少 3 个抽屉。
- 抽屉能开合，并且不会脱离卡片盒。
- 有至少 3 张测试卡片。
- 卡片是有厚度的 3D 物体。
- 卡片正面有占位贴图或导入贴图。
- 卡片可以被选中。
- 卡片可以拿起和移动。
- 卡片可以旋转。
- 卡片可以翻面。
- 卡片可以放到桌面。
- 卡片可以放进抽屉。
- 卡片可以从抽屉中取出。
- 有简单分隔片。
- 分隔片可以被拿起、移动、放下和放入抽屉。
- 可以导入本地图片生成新卡片。
- 可以保存和加载世界状态。
- 文档说明了大量卡片优化策略。
- 没有 OCR、搜索、标签、自动分类、自动链接、知识图谱、AI 摘要。

## 常见故障

- 看不到对象：按 `1` 或 `Home` 回到总览视角。
- 抽屉无法放入卡片：先把抽屉拉开，再把卡片拖到抽屉内部区域。
- 卡片拖出抽屉后又回到抽屉：松开位置仍在抽屉判定区域内，把卡片拖远再松开。
- 导入图片不显示：确认文件格式为 `png`、`jpg`、`jpeg` 或 `webp`。
- 状态没有恢复：先保存一次；运行时状态文件位于 Godot `user://world_state.json`。
- 按键没有反应：先点击并选中对应对象，或确认鼠标不在 UI 控件上。
