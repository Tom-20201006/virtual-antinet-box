# PROJECT_CHARTER

## 最高原则

计算机只模拟物理卡片盒，不替用户思考。

VirtualAntinetBox 只负责提供虚拟桌面、卡片盒、抽屉、卡片、分隔片、碰撞边界、拿取、放置、插入、抽出、翻面、旋转、桌面摊开、状态保存和状态恢复。

## 允许功能

- 3D 桌面、卡片盒、抽屉、分隔片、卡片。
- 本地图片作为卡片正面或背面贴图。
- 鼠标选择、拖拽、放置。
- 键盘旋转、翻面、保存、加载、视角切换。
- 抽屉沿固定轴开合，并保存开合程度。
- 保存卡片 ID、资源路径、位置、旋转、所在区域、尺寸、简单物理状态。
- 为大量卡片预留对象池、LOD、缩略图和静止冻结策略。

## 禁止功能

以下功能不得实现，也不得作为当前流程的默认入口或预留流程出现：

- OCR。
- 搜索。
- 标签。
- 自动分类。
- 自动链接。
- AI 摘要。
- 知识图谱。
- Markdown 笔记编辑器。
- 根据卡片内容生成编号。
- 根据卡片内容推荐位置。
- 数据库式知识管理。
- 云同步。
- 用户账号。
- 网页剪藏。
- 自动索引。
- 任何根据卡片文字内容进行智能处理的功能。

## 后续开发约束

后续可以改进模型、材质、手感、性能和导入流程，但必须保持“物理状态与资源路径”这一保存边界。任何内容理解、语义组织或知识管理功能都应放在本项目之外。
