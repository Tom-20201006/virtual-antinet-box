# Acceptance

本文档定义当前阶段的验收标准。详细步骤以 `docs/validation.md` 为准。

## 最低可接受标准

- Godot 项目能打开。
- 主场景能运行。
- 有完整桌面。
- 有卡片盒。
- 有至少 3 个抽屉。
- 抽屉能开合。
- 有至少 3 张测试卡片。
- 卡片有厚度和碰撞体。
- 卡片可以被选中、拿起、移动、旋转、翻面。
- 卡片可以放桌面和放入抽屉。
- 有分隔片。
- 可以导入本地图片生成新卡片。
- 可以保存和加载世界状态。
- 没有 OCR、搜索、标签、自动分类、自动链接、AI 摘要、知识图谱。

## 功能完整标准

- 抽屉开合稳定，不会脱离卡片盒。
- 卡片放入抽屉时有简化对齐辅助。
- 卡片从抽屉取出后能放到桌面摊开区域。
- 导入图片只作为贴图显示，不做文字识别。
- 保存数据只包含物理状态和资源路径。

## 交互体验标准

- 摄像机可旋转、平移、缩放。
- 可快速回到桌面总览。
- 可聚焦当前卡片或抽屉。
- 被选中的卡片、分隔片或抽屉有视觉提示。
- 操作不应导致严重穿模、抖动或乱飞。

## 构建或导出标准

当前阶段不要求导出桌面应用包。最低要求是 Godot 编辑器内可运行，且命令行 headless 加载无脚本解析错误。

## 文档标准

- `README.md` 说明项目目标、运行方式、功能和限制。
- `PROJECT_CHARTER.md` 明确最低限度数字化原则。
- `docs/validation.md` 提供详细逐步验收。
- `docs/TESTING.md` 说明测试方式。
- `docs/COMMAND_QUEUE.md` 是后续命令主入口。
- `AGENTS.md` 明确 Codex 每轮执行规则。

## 当前阶段不纳入验收

- 精细木纹。
- 真实纸张弯曲。
- 手部模型。
- VR。
- 云同步。
- 用户账号。
- 搜索、标签、OCR、AI 摘要、知识图谱等语义功能。
