# Performance Strategy

## 当前已实现

- 卡片不使用长期活跃的高成本 `RigidBody3D`。
- 拖拽时由鼠标平面控制，放下后标记为 sleeping。
- `PhysicsSleepManager` 记录所有交互对象，当前操作对象 active，其余对象 sleeping。
- 抽屉使用受限运动，只沿固定轴移动，有最小和最大位置。
- 抽屉内卡片使用简化排列和吸附，不依赖复杂实时物理堆叠。
- 场景结构把外观、碰撞和交互分离，方便以后替换模型或降级显示。

## 5000 到 20000 张卡片扩展方向

- 建立对象池：只实例化当前可见或可交互卡片，其余卡片保存为轻量状态记录。
- 抽屉深处卡片使用静态堆叠表示，拉近或翻找时再实例化为可交互卡片。
- 为图片建立缩略图和高清图两级资源：远处使用缩略图，近看时加载高清贴图。
- 对卡片网格做 LOD：远处使用简单盒体，近处使用完整正反面材质。
- 抽屉内容按可见窗口分页，不让整抽屉全部卡片同时参与碰撞。
- 保存文件仍保持物理状态与资源路径，不引入数据库式知识管理。

## 不采用的优化方向

- 不通过搜索、标签或内容索引来提高定位效率。
- 不根据文字内容推断卡片位置。
- 不引入云同步或账号系统作为性能方案。
